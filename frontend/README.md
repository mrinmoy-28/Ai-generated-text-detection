## Overview

The AI Text Detector frontend provides an intuitive, professionally-designed interface for analyzing text and documents to determine whether they were written by humans or generated by AI models. Whether you're a student checking assignments, an educator verifying authenticity, or a researcher studying AI output, this tool offers detailed insights with actionable results.

**Key Highlights:**
- ⚡ Processes documents in seconds
- 🎯 4-model ensemble analysis for superior accuracy
- 📦 Real-time batch processing capabilities
- 📊 Beautiful analytics dashboard with visualizations
- 🎨 Professional, modern UI with smooth animations
- 📱 Fully responsive design (mobile, tablet, desktop)
- 🔍 Submission history tracking for compliance
- 🔐 Secure and private text analysis

## Core Features

### 🔍 Text Detection
Analyze any text quickly and accurately. Simply paste your content or upload a file, and receive instant results with confidence scores. The system uses a multi-model approach to ensure reliable and balanced detection.

**Supported formats:** .txt, .pdf, .docx files up to 10MB each

**Features:**
- Real-time word counter
- Confidence circle visualization
- Model-by-model breakdown
- Instant feedback with beautiful UI

### 🔄 Document Comparison
Compare two documents side by side to identify similarities and differences in AI-generated content. Useful for plagiarism detection and content analysis.

**Features:**
- Dual input areas (text or files)
- Similarity percentage display
- Side-by-side verdict comparison
- Detailed difference metrics

### 📦 Batch Processing
Process multiple documents simultaneously with real-time progress tracking. Ideal for educators grading assignments or researchers analyzing large datasets.

**Features:**
- Drag & drop upload interface
- Real-time progress bar (0-100%)
- File list with metadata
- Summary statistics
- Results export capability

### 📊 Advanced Analysis
Understand *why* the system made its decision. The explainability feature breaks down which specific words and phrases influenced the AI detection verdict.

**Includes:**
- Individual model scores (Statistical, RoBERTa, Zero-Shot, Watermark)
- Sentence-level highlighting of AI-generated content
- Word-level explainability using SHAP methodology
- Confidence distribution metrics

### 📈 Analytics Dashboard
Track detection trends over time with beautiful visualizations and interactive stat cards. View summary statistics and detection patterns.

**Displays:**
- Total scans processed
- AI vs Human percentages
- Average confidence scores
- Detection trends
- Professional stat cards with hover effects

### 📋 Submission History
Never lose track of your analyses. The history feature maintains a searchable, filterable log of all submissions with color-coded results.

**Features:**
- Sortable/filterable table
- Pagination (20 per page)
- Color-coded badges
- Timestamp tracking
- Quick search

## API Integration

The frontend communicates with a backend API. Ensure your backend is running at `http://localhost:8000`.

### Available Endpoints

**Detection Operations**
- `POST /detect` - Analyze text for AI generation
- `POST /detect/sentences` - Break down detection by sentence
- `POST /detect/file` - Analyze uploaded documents
- `POST /detect/batch` - Process multiple files

**Advanced Features**
- `POST /compare` - Compare two text inputs
- `POST /compare/files` - Compare uploaded documents
- `POST /analyze/advanced` - Deep analysis with additional metrics
- `POST /plagiarism` - Plagiarism detection analysis

**Reports and Data**
- `POST /explain` - SHAP explainability analysis
- `POST /report` - Generate downloadable PDF reports
- `POST /export` - Export results as PDF or DOCX
- `GET /history` - Retrieve past submissions
- `GET /stats` - Retrieve analytics data
- `GET /health` - Check backend API status
- `GET /status/{jobId}` - Check processing status

### Configuration

To change the backend URL, edit `src/api/detector.js`:

```javascript
const API_BASE_URL = 'http://localhost:8000'
```

You can also set the URL via environment variables in a `.env` file:

```
VITE_API_BASE_URL=http://localhost:8000
```

## Troubleshooting

### Backend Not Connecting

**Problem:** "Failed to connect to backend" error

**Solutions:**
- Verify the backend API is running at `http://localhost:8000`
- Check that the correct backend URL is configured
- Verify firewall settings allow local connections
- Check browser console for CORS-related errors

### File Upload Issues

**Problem:** Files not uploading or processing

**Solutions:**
- Ensure file size is under 10MB
- Verify file format is supported (.txt, .pdf, .docx)
- Check that backend has file upload handlers enabled
- Verify disk space is available

### Blank Page on Load

**Problem:** Application shows blank/white page

**Solutions:**
- Clear browser cache and cookies
- Hard refresh the page (Ctrl+Shift+R or Cmd+Shift+R)
- Check browser developer console for JavaScript errors
- Verify all dependencies installed: `npm install`
- Try deleting `node_modules` and reinstalling

### Build Errors

**Problem:** Errors when running `npm run build`

**Solutions:**
```bash
# Clear cache and reinstall dependencies
rm -rf node_modules package-lock.json
npm install

# Try building again
npm run build
```
